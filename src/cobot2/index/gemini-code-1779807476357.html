프로젝트 개요

본 프로젝트는 생산 현장의 자동화와 관제 고도화를 위해 구축된 협동 로봇 기반 디지털 트윈 관제 시스템입니다.

두산 로보틱스 협동 로봇이 멀티 워크스테이션을 순회하며 제품을 스캔하고, YOLOv11x 엔진을 활용해 나사/볼트의 단차 결함을 정밀 검출합니다. 최종적으로 연산된 3D 기하학 데이터를 웹 환경에 동기화하여 작업자가 실시간으로 모니터링할 수 있는 프레임워크를 개발했습니다.

핵심 트러블슈팅 (Troubleshooting)

⚡ 1. 3D 데이터 스트리밍 웹 병목 및 브라우저 렉 현상
문제점
실시간으로 대용량 점구름(Point Cloud)을 갱신 및 전송 시, 프론트엔드 렌더링 부하와 Firebase DB 트래픽 동시 과부하로 심각한 웹 화면 버벅임 및 데이터 누락 발생.
해결방법
연속 실시간 갱신 구조에서 탈피하여 로봇의 스캔 완료 플래그(3d_scan_done=True)를 기점으로 스냅샷 데이터를 호출하는 '캡처형 아키텍처'로 전환. 배경 점구름을 상황별(1만~5만 점)로 조절하는 가변 샘플링 레이트를 도입해 리소스 최적화 성공.
[시연화면] 3D 렉 완화 구조 & 샘플링 제어 대시보드 (dashboard_performance.png)
🔄 2. 3D 그래픽스 차원 왜곡 및 마우스 회전 불가능 조작 제약
문제점
웹 화면 초기 로드시 입체 뷰가 아닌 2D 평면 스타일의 축으로 왜곡되어 노출되었고, 마우스 드래그 조작 시 입체 회전이 아닌 단순 수평 이동(Pan)만 지원되어 나사 결함의 입체적 판독 불가능.
해결방법
렌더링 파이프라인 유입단에 type: 'scatter3d' 파라미터를 명시하여 2D 왜곡 원인을 제거하고, 대시보드 레이아웃의 기본 모드를 회전 모드(dragmode: 'rotate')로 변경. 초기 뷰포트에 상단 대각선 45도 최적 앵글(scene.camera.eye)을 강제 주입해 안정적인 탑다운 입체 뷰 확보.
[비교화면] 2D 평면 왜곡 해결 및 상단 45도 최적 입체 뷰 (3d_viewport_angle.png)
👁️ 3. 가혹한 공장 환경에서의 시인성 저하 및 결함 인지 한계
문제점
고해상도 배경 점구름이 화면에 촘촘하게 깔려, AI가 판정한 결함 유무 플래그 마커와 정보 라벨 텍스트가 배경에 파묻혀 원거리 작업자가 직관적으로 불량을 식별하기 어려운 한계 봉착.
해결방법
공장 현장 오퍼레이터의 작업 환경을 고려하여 시인성 특화 캘리브레이션 진행. 결함 마커 스케일을 대폭 확장하고(MARKER_SIZE=18), 가혹한 조명 아래에서도 판독이 용이하도록 고대비 색상 매핑 및 텍스트 대형화 레이아웃을 전면 적용하여 시각적 직관성 극대화.
