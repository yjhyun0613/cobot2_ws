System Pipeline

01

ROS 2 Trigger 서비스

로봇이 5개 작업대에 순차 배치 → /vision_inspect Trigger 비동기 호출로 검사 시작

02

YOLO 비전 스캔

YOLOv11x로 나사 존재 판정. 미발견 시 SKIP → 저장·통신 연산 원천 차단하여 자원 절약

03

핀홀 카메라 3D 복원

CameraInfo(fx,fy,cx,cy) + Aligned Depth → 역투영 공식으로 정밀 3D 좌표 복원 → TF2 base_link 변환

04

동적 도넛 알고리즘

RANSAC 평면 피팅 → HSV 나사 추출 → 코어 최고점 vs 철판 중앙값의 상대 단차(mm) 연산 → 10mm 초과 시 불량

05

Firebase 클라우드 동기화

5개 모듈화 함수로 실시간 DB 분산 저장: live_scan, inspection_records, twin_states, indexes, events

06

3D 웹 대시보드

Plotly.js 3D Scatter + Firebase 웹소켓으로 지연 없는 실시간 원격 모니터링. 작업대 5개 드롭다운 전환

Key Challenges & Solutions

Phase 1~2. Open3D 탈피 및 스냅샷 전환 (v00~v05)

문제

Open3D 독점 스레드 → 웹 연동 불가. 실시간 대용량 PC 유입 → 메모리 오버플로우 및 연산 지연

해결

Plotly HTML 자동 생성으로 웹 독립 구동 확보. Open3D → 순수 NumPy 고속 복셀 다운샘플링 전환. '정지 순간 확정형 스냅샷' 패러다임 + 0.2m~1.2m 거리 윈도우 필터

Phase 3. ROS 2 TF2 표준 좌표계 확립 (v06~v09)

문제

수동 FK 기구학 계산 시 링크 가공 오차·카메라 장착 공차로 공간적 비틀림 및 스케일 미스매칭 누적

해결

ROS 2 TF2(TransformListener) 완전 전환 + 두산 커널 API(GetCurrentPosx) + StaticTransformBroadcaster로 카메라 오프셋(Y 75mm, Z 39.91mm) 및 경사각 보정

Phase 4. YOLO AI 융합 및 3D 기하 복원 (v10~v12)

문제

2D 박스→3D 점구름 비율단 매핑 시 해상도 비대칭 인덱스 오류. 금속 반사로 뎁스 센서 NaN(도넛 현상) → 시스템 정지

해결

11×11 윈도우 매트릭스 NaN 주변 탐색 필터. Pinhole 카메라 역투영 공식 x=(u-cx)×z/fx 직접 구현 → 물리 3D 좌표 복원 성공

Phase 5. Zero-Defect 판정 혁신 및 클라우드 최적화 (v13~v22)

문제

YOLO 소프트맥스만 의존 → 조명 난반사 오판. 대용량 HTML 통째 업로드 → 대역폭 정체 및 스토리지 한계

해결

YOLO를 위치 가이드로 한정 → "동적 도넛 알고리즘"으로 상대 단차 수학적 연산. JS 정적 변수 모듈 분할 업로드로 페이로드 85%+ 압축

Phase 6. 멀티 워크스테이션 5개 작업대 순회 (multi)

문제

단일 노드로 5개 작업대 순회 시 데이터 덮어쓰기 및 오케스트레이션 제어 불가

해결

ROS 2 Trigger 서비스 구축 → 나사 미발견 자동 Skip → section_name 기반 DB 파티셔닝 분산 저장. 비동기 모듈러 메서드 체인 완성

Phase 7~8. 센서 동기화 · 자율 세션 관리 (v10_03)

문제

RGB/Depth/PC 타임 Skew로 매칭 오차. SDK 업데이트 시 토픽 변경 → 배포 실패. 유휴 후 세션 오염

해결

QoS BEST_EFFORT + RANSAC + HSV 블랙 마스크 + 3단계 검증 → 0.5mm 정밀 보정. 토픽 자동 스캔 동적 구독. 60초 타임아웃 reset_session() 자동 초기화
